\begin{abstract}
The \textbf{AudioToLights} project explores Sonic Interaction Design through an automated system capable of running a real-time synchronization between an audio track execution (MIDI file) and a light show. By leveraging Generative Artificial Intelligence (Gemini 2.5 Flash), the system produces complex visual interpretations of musical parameters, interfacing with the QLC+ software via the OSC protocol.
\end{abstract}

\section{Introduction}
The dynamic and synchronized control of lighting in a performative environment is a key element of Sonic Interaction Design. The objective of the project is to automate the work of a Light Jockey by pre-analyzing musical events to generate lighting choreographies. To achieve this, a Python program has been developed, consisting of several modules ranging from MIDI parsers to OSC communication.

\section{System Architecture}
The software architecture is divided into five main modules:
\begin{itemize}
    \item \textbf{MIDI Parser (\texttt{MidiParser}):} Reads and analyzes MIDI files (.mid) extracting note events (Pitch, Velocity, and Rhythm).
    \item \textbf{AI Analyzer (\texttt{AIAnalyzer}):} Queries the LLM model (Gemini 2.5 Flash) providing the extracted MIDI events to generate a creative lighting mapping in the desired JSON format.
    \item \textbf{Mapper:} Regulates the conceptual assignment of the instructions provided by the AI to the active lighting equipment (e.g., PAR, Moving Head, Strobe).
    \item \textbf{OSC Sender (\texttt{OSCSender}):} Manages communication via the OSC (Open Sound Control) protocol, sending signals to a local instance of QLC+ running on port 7700.
    \item \textbf{Playback Engine (\texttt{PlaybackEngine}):} Dedicated to the time synchronization and parallel playback execution of the audio (via \textit{pygame}) and of the generated lighting commands.
\end{itemize}

\section{Artificial Intelligence Integration}
The AI-based approach allows aesthetic and compositional results that are superior to simple fixed "rule-based" algorithms. A structured prompt containing the chronological list of events and the fixtures available in the rig is passed. The output generated by Gemini modulates the intensity based on the \textit{velocity} of the note and assigns light types to different rhythmic and \textit{pitch} variations.

\section{Conclusion and Future Developments}
The \textit{AudioToLights} system demonstrates the feasibility of a smoothed and synchronized auto-generated light-show. Possible future developments include support for real-time audio analysis (data directly from a microphone or pre-recorded non-MIDI tracks) and device management via the Art-Net standard for extended physical control of the hardware.
